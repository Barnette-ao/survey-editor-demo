# Creator 文件夹代码分析报告

根据对 `src/views/creator` 文件夹的代码分析,这是一个**问卷调查系统的设计器模块**,主要功能是让用户可视化地创建、编辑和预览问卷。以下是详细分析:

---

## 一、整体架构

这个模块采用 **Vue 3 + SurveyJS** 框架构建,主要分为以下几个部分:

1. **设计器界面** (design.vue) - 用户创建和编辑问卷
2. **预览界面** (preview.vue) - 实时预览问卷效果
3. **逻辑设置** (LogicSettingDialog.vue) - 配置题目间的跳转和显示逻辑
4. **配置管理** (config/) - 处理数据的增删改查和逻辑规则
5. **题型组件** (components/) - 各种题型的渲染组件

---

## 二、核心功能模块

### 1. 题型支持 (questionTypeList.js)

系统支持多种题型,分为5大类:

- **选择题**: 单选、多选、下拉、量表、选图片
- **填空题**: 单行填空、多行填空、多项填空
- **打分排序**: 评价、打分、排序
- **备注说明**: HTML说明、分页
- **其他**: 文件上传、签名、矩阵单选

### 2. 题目管理 (handleElementAndPage.js)

核心功能包括:

- `addQuestionElement()` - 添加新题目
- `deleteQuestion()` - 删除题目(同时删除相关逻辑规则)
- `getSwitchTargetElement()` - 题型切换(如单选转多选)
- `handleCopyElement()` - 复制题目
- `addPage()` / `deletePage()` - 页面管理

关键设计:

```javascript
// 每个题目都有唯一ID和name
{
  id: uuidv4(),           // 唯一标识
  name: `Q${number + 1}`, // 题目编号
  type: "radiogroup",     // 题型
  title: "题目标题",
  choices: [...]          // 选项
}
```

### 3. 逻辑规则系统 (updateLogic.js, parsingLogic.js)

这是系统的核心功能,支持两种逻辑:

#### 跳转逻辑 - 根据答题情况跳转到指定题目:

```javascript
{
  ifConditions: [{
    elementId: "题目ID",
    state: "selected",      // 选中/未选中/已答等
    choiceIndex: 0          // 选项索引
  }],
  thenCondition: {
    action: "jump",
    targetElementId: "目标题目ID"
  }
}
```

#### 显示逻辑 - 根据条件显示/隐藏题目:

```javascript
{
  ifConditions: [...],
  thenCondition: {
    action: "show",
    targetElementId: "目标题目ID"
  }
}
```

#### 逻辑表达式示例:

- 选中某选项: `{Q1} = '选项1'`
- 未选中: `{Q1} != '选项1'`
- 已答: `{Q1} notempty`
- 分数比较: `{Q1} > 3`

### 4. 拖拽功能 (dragElementOrPage.js)

使用 Sortable.js 实现:

- 题目拖拽排序
- 页面拖拽重组
- 跨页面拖拽题目
- 自动更新题号

关键处理:

```javascript
// 处理视图索引和数据索引的差异
// 第一页有简介元素,视图索引需要+1
if (questionSettings.pages.length === 1) {
  oldIndex += 1
}
```

### 5. 数据持久化 (index.js, helpers.js)

- `loadSettingsFromDatabase()` - 从数据库加载问卷配置
- `updateDefaultSettings()` - 保存到数据库
- `beforeSaveToDatabase()` - 保存前数据格式化
- `afterGetInitialSettings()` - 加载后数据格式化

数据转换处理:

```javascript
// 保存前: 将rating子类型统一为rating
ratinglabel/ratingstars/ratingsmileys → rating

// 加载后: 为每个元素添加id和number
element → { id: uuid(), number: 1, ...element }
```

## 四、关键技术点

1. **响应式数据管理**: 使用 Vue 3 的 reactive/ref 管理问卷配置
2. **深拷贝处理**: 使用 lodash 的 cloneDeep 避免引用问题
3. **UUID生成**: 使用 uuid v4 生成唯一标识
4. **正则表达式**: 解析逻辑表达式
5. **函数式编程**: 使用 pipe 组合函数处理数据流

```
用户操作 → 组件事件 → 更新 questionSettings →
```

## 七、核心流程详解

### 7.1 添加题目流程

1. 用户点击题型按钮
2. 调用 `addQuestionElement(questionSettings, elementType, selectedQuestionId)`
3. 通过 `createNewElement()` 创建新题目对象
4. 根据选中状态决定插入位置
5. 调用 `formattedNumber()` 更新所有题号
6. 滚动到新题目位置

### 7.2 逻辑规则设置流程

1. 用户打开逻辑设置对话框
2. 配置如果条件(ifConditions)和则条件(thenCondition)
3. 调用 `handleLogicRulesUpdate()` 处理规则更新
4. 通过 `setLogicRule()` 将规则转换为表达式
5. 更新 questionSettings 中的 triggers 或 visibleIf 属性

### 7.3 拖拽题目流程

1. 用户拖拽题目
2. Sortable.js 触发 onEnd 事件
3. 计算 fromPageIndex 和 toPageIndex
4. 调用 `dragQuestionElement()` 更新数据
5. 调用 `formattedNumber()` 重新排序题号
6. 重新渲染页面

### 7.4 数据保存流程

1. 用户编辑问卷
2. 触发自动保存或手动保存
3. 调用 `beforeSaveToDatabase()` 格式化数据
4. 调用 `UpdateBusQuestionJson()` 保存到数据库
5. 返回保存结果

## 八、重要设计模式

### 8.1 组件映射模式

通过对象映射实现动态组件加载,避免大量 if-else 判断:

```javascript
const component = componentMap[element.type]
```

### 8.2 管道模式 (Pipe)

使用函数组合处理数据转换:

```javascript
const pipe = (...fns) => (x) => fns.reduce((result, f) => f(result), x);

const processElements = pipe(
  arr => arr.map(checkElementType),
  arr => arr.filter((element) => element !== null),
  arr => arr.map((element) => pipe(addId, addNumber)(element)),
)
```

### 8.3 策略模式

根据不同的逻辑类型执行不同的处理策略:

```javascript
switch (action) {
  case 'jump':
    handleJumpCondition(...);
    break;
  case 'show':
    handleShowCondition(...);
    break;
}
```
