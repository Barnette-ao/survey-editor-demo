Design 页中「必须可撤销」的操作清单（MVP）

这些是用户强烈预期能撤销的行为：

一、结构性操作（最高优先级）

这是核心。

1️⃣ 新增题目

addQuestion

addPage（如果支持）

👉 必须可 undo
否则用户会非常痛苦。

2️⃣ 删除题目 / 删除页面

deleteQuestion

deletePage

👉 必须可 undo
这是“灾难级误操作”。

3️⃣ 拖拽排序（题目 / 页面）

reorderQuestions

reorderPages

👉 必须可 undo
而且：

一次拖拽 = 一个原子操作

不拆成多步

你之前已经意识到这一点，非常好。

二、题目类型变更（结构级）
4️⃣ 修改题目类型

例如：

input → radio

rating → dropdown

这种通常会：

改 schema

清空部分配置

👉 必须 undo

这是隐性破坏性操作。

三、关键属性修改（配置级）

不是所有字段都要第一期支持 undo。

但这几个必须：

5️⃣ title / label / description

用户最频繁改的：

问题标题

描述文案

👉 必须 undo

6️⃣ choices / options

单选、多选的选项：

add choice

delete choice

reorder choice

edit choice text

👉 必须 undo

否则基本不可用。

四、整题级操作（复合）
7️⃣ 复制题目

copyQuestion

本质是：

insert + deep clone

👉 必须 undo

8️⃣ 批量操作（如果有）

例如：

多选删除

多选移动

👉 当作一次原子操作

第一阶段可以不支持 undo 的

你现在完全可以明确排除：

🚫 UI 状态类

是否展开

是否选中

hover

临时高亮

这些是 UI 状态，不进 Draft。

🚫 面板开关

属性面板开关

JSON 面板开关

🚫 输入中的瞬时状态

例如：

正在输入第 3 个字符

debounce 后才进 undo。
