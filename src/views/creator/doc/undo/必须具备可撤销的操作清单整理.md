```js
    // 原操作：删除原页码 + 在目标页码前插入空页面
    // 撤销操作：删除插入的空页面 + 恢复原页码
    
    // 1. 删除在 originalToPageIndex - 1 位置插入的空页面
    handleDeletePage(questionSettings, questionSettings.value.pages[originalToPageIndex - 1], originalToPageIndex - 1)
    
    // 2. 在原位置 originalFromPageIndex 恢复页码
    // 需要调用 addEmptyPageBeforePage 在原位置重新创建页面
    addEmptyPageBeforePage(questionSettings, originalFromPageIndex)
    
    // 3. 重新排序题目编号
    formattedNumber(questionSettings.value)
}
```

## 结论

1. **undoDragQuestionElement**: 可以直接复用现有的 `dragQuestionElement` 函数，只需要对调参数位置
2. **undoDragPageToPage**: 可以组合使用现有的 `handleDeletePage` 和 `addEmptyPageBeforePage` 函数
3. **undoDragPageToElement**: 最复杂，建议重新实现专门的撤销函数，因为涉及复杂的页面结构恢复

## 建议
对于页码拖拽的撤销操作，建议在执行拖拽操作时保存更详细的状态快照，包括：
- 原页面的完整元素列表
- 页面分割的具体位置
- 受影响页面的原始结构

这样可以更准确地进行撤销操作。
