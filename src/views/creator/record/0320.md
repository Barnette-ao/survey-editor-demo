## 修改点
不会让所有的逻辑规则一起被设置。因此保存按钮的插入规则是要改的。不会在遍历整个logicRules，只会设计一个logicRule

## 修改数据结构
因为每个题目都单独设计自己的逻辑规则，无论跳转逻辑还是显示逻辑。任一题目不能再获得所有题目的逻辑规则了。那么questionSettings.logicRules这个存储了所有题目的逻辑规则的属性，用这个属性来实现插入新的逻辑规则，删除逻辑规则，修改逻辑规则，实现这些操作，会变得很繁琐。

可以在每个题目元素中添加logicRule这个属性来代替在questionSettings中添加logicRules属性来代替这个设计

这样就完全不用

questionSettings = {
  logicRuleMap: {
    "rule1": {
      id: "rule1",
      ifConditions: [
        { elementId: "element2", state: "selected" },
        { elementId: "element3", state: "answered" }
      ],
      thenConditions: [
        { targetElementId: "element1", action: "show" }
      ]
    }
  }
}
