因为DOM节点的更新只有刷新一次才能完成，就是下一次生命周期启动的时候，才会刷新一遍。如果改了dom节点的内容。

而<p id = "test">{{text}}</p>

	})

由此可知，在某个函数，无论是什么函数，如果修改了DOM节点的值，然后打印其值，DOM节点的值还会是旧值，不会是最新的值。

这就像是在template中使用{{text}}，当text的数据类型是原始数据类型时，不会出现延迟更新，但如果是对象，那么就会出现这种延迟更新。如果text是对象，

总结，v-if或者v-show如果是由函数决定返回值，函数的参数的数据类型可以是原始数据类型，不能是对象数据类型

对象数据类型的参数，可以用计算属性来替换，用计算属性返回一个函数，该函数的参数是原始值数据类型，然后根据原始数据类型，获取该对象数据类型的值，这样还可以做到及时更新
