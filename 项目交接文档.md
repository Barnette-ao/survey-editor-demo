# 问卷编辑器系统 - 项目交接文档

## 项目概述

### 基本信息
- **项目名称**: 问卷编辑器系统 (questionnaire-demo)
- **技术栈**: Vue 3 + Vite + TypeScript + Element Plus + SurveyJS
- **开发模式**: SPA (单页应用)
- **状态管理**: Pinia
- **构建工具**: Vite
- **UI框架**: Element Plus
- **编辑器**: Monaco Editor + TipTap

### 项目特点
- 🎯 **可视化问卷编辑器**: 支持拖拽式问卷设计
- 🔄 **复杂逻辑规则系统**: 支持跳转逻辑和显示逻辑
- 📝 **多种题型支持**: 单选、多选、评分、文本等多种题型
- 🎨 **实时预览**: 设计、预览、JSON编辑三种模式
- 💾 **数据持久化**: 支持本地存储和API接口
- 🧩 **Composable架构**: 使用Vue 3 Composition API重构

## 核心功能模块

### 1. 问卷编辑器 (`src/views/creator/`)

#### 主要组件
- **`index.vue`**: 编辑器主入口，包含三个标签页
  - 项目设计: 可视化编辑界面
  - 预览: 问卷预览模式
  - JSON编辑器: 直接编辑JSON配置

- **`design.vue`**: 核心设计界面
  - 题型拖拽面板
  - 问卷画布区域
  - 属性设置面板

- **`preview.vue`**: 问卷预览组件
- **`jsonEditor.vue`**: JSON编辑器组件

#### 题型组件 (`components/`)
支持多种题型，每种题型都有对应的组件和设置面板：
- `radiogroup.vue` - 单选题
- `checkbox.vue` - 多选题
- `dropdown.vue` - 下拉选择
- `scale.vue` - 量表题
- `score.vue` - 评分题
- `ranking.vue` - 排序题
- `matrixRadio.vue` - 矩阵单选
- `sigleText.vue` - 单行文本
- `multipleText.vue` - 多行文本
- `fileupload.vue` - 文件上传
- `sign.vue` - 签名题

### 2. 逻辑规则系统

#### 核心功能
- **跳转逻辑**: 根据答题情况跳转到指定题目
- **显示逻辑**: 根据条件显示或隐藏题目
- **条件设置**: 支持多条件组合（且/或）
- **规则验证**: 确保逻辑规则的完整性

#### 关键文件
- `LogicSettingDialog.vue` - 逻辑设置对话框
- `composables/useLogicRule/` - 逻辑规则相关的Composable
- `config/updateLogic.js` - 逻辑更新处理

### 3. Composable架构 (`composables/`)

项目采用Composition API重构，主要Composable包括：

#### 核心业务逻辑
- `useCurrentElement.ts` - 当前选中元素管理
- `useElementOperations.ts` - 元素操作（增删改）
- `useLogicRules.ts` - 逻辑规则管理
- `useQuestionCreation.ts` - 题目创建逻辑

#### UI状态管理
- `useSelectionState.ts` - 选择状态管理
- `useHoverState.ts` - 悬停状态管理
- `useSettingPanelState.ts` - 设置面板状态

#### 数据处理
- `useElementMutation.ts` - 元素数据变更
- `useChoiceSetting.ts` - 选项设置管理
- `useSurveyValidation.ts` - 问卷验证

### 4. 配置系统 (`config/`)

- `componentAndSettingMap.js` - 组件与设置面板映射
- `dragElementOrPage.js` - 拖拽元素配置
- `handleElementAndPage.js` - 元素和页面处理
- `parsingLogic.js` - 逻辑解析
- `updateLogic.js` - 逻辑更新

## 数据流架构

### 状态管理
使用Pinia进行状态管理，主要Store：
- `editorStore.js` - 编辑器状态管理

## 核心业务逻辑

### 1. 问卷数据结构
```typescript
interface QuestionSettings {
  pages: Page[]           // 页面数组
  logicRules?: LogicRule[] // 逻辑规则
  title: string           // 问卷标题
  description?: string    // 问卷描述
}

interface QuestionElement {
  id: string             // 唯一标识
  type: string          // 题型
  title: string         // 题目标题
  choices?: Choice[]    // 选项（选择题）
  isRequired?: boolean  // 是否必填
  // ... 其他属性
}
```

### 2. 逻辑规则系统
```typescript
interface LogicRule {
  id: string
  ifConditions: IfCondition[]  // 如果条件
  thenCondition: ThenCondition // 那么动作
}

interface IfCondition {
  elementId: string    // 关联元素ID
  state: string       // 状态（选中/未选中等）
  choiceIndex?: string // 选项索引
  connector?: 'and' | 'or' // 连接符
}
```

### 3. 组件映射机制
通过配置文件实现题型组件的动态加载：
```javascript
// componentAndSettingMap.js
export const componentMap = {
  'radiogroup': () => import('../components/radiogroup.vue'),
  'checkbox': () => import('../components/checkbox.vue'),
  // ... 其他组件映射
}
```

## 已知问题与注意事项

### 1. 技术债务
- `LogicSettingDialog.vue` 组件过于复杂（500+行）
- 部分业务逻辑与UI逻辑耦合
- 需要继续推进Composable重构

### 2. 性能考虑
- 大量题目时可能出现性能问题
- 复杂逻辑规则计算耗时
- 建议实施虚拟滚动和分页加载

### 3. 浏览器兼容性
- 主要支持现代浏览器
- IE不支持（使用了ES6+特性）
- 移动端适配需要进一步优化

### 4. 数据安全
- 本地存储数据未加密
- API接口需要添加认证机制
- 敏感数据需要脱敏处理
