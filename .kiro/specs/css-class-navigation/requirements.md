# CSS 类名跳转功能需求文档

## 介绍

为 Kiro IDE 添加 CSS 类名跳转功能，让开发者能够快速从 Vue 模板中的类名跳转到对应的样式定义。

## 术语表

- **CSS 类名**: HTML/Vue 模板中 class 属性的值，如 `class="logic-header"`
- **样式定义**: CSS/SCSS 中的选择器规则，如 `.logic-header { ... }`
- **跳转**: 光标自动定位到目标代码位置
- **Vue 单文件组件**: 包含 `<template>`、`<script>`、`<style>` 的 .vue 文件

## 需求

### 需求 1: 基本类名跳转

**用户故事:** 作为开发者，我希望能够从模板中的类名快速跳转到样式定义，以便快速查看和修改样式。

#### 验收标准

1. WHEN 用户在 Vue 模板中按住 Ctrl/Cmd 并点击类名 THEN 系统应该跳转到同一文件 `<style>` 部分对应的样式定义
2. WHEN 类名在样式中不存在 THEN 系统应该显示"未找到样式定义"的提示
3. WHEN 用户点击的不是有效类名 THEN 系统应该不执行任何跳转操作

### 需求 2: 多类名支持

**用户故事:** 作为开发者，我希望能够跳转到多个类名中的任意一个，以便灵活查看不同的样式。

#### 验收标准

1. WHEN 元素包含多个类名（如 `class="btn primary large"`）THEN 用户应该能够点击任意一个类名进行跳转
2. WHEN 点击的类名存在样式定义 THEN 系统应该跳转到对应的样式
3. WHEN 点击的类名不存在样式定义 THEN 系统应该显示相应提示

### 需求 3: 样式定位准确性

**用户故事:** 作为开发者，我希望跳转能够精确定位到样式规则的开始位置，以便立即开始编辑。

#### 验收标准

1. WHEN 跳转到样式定义 THEN 光标应该定位在选择器行（如 `.logic-header` 行）
2. WHEN 样式规则有多行 THEN 光标应该定位在选择器的开始位置
3. WHEN 存在嵌套样式（SCSS）THEN 系统应该能够正确定位到对应的嵌套层级

### 需求 4: 视觉反馈

**用户故事:** 作为开发者，我希望在悬停类名时能够看到视觉提示，以便知道该类名是否可以跳转。

#### 验收标准

1. WHEN 用户悬停在可跳转的类名上 THEN 类名应该显示下划线或其他视觉提示
2. WHEN 用户悬停在不可跳转的类名上 THEN 不应该显示跳转提示
3. WHEN 用户按住 Ctrl/Cmd 键悬停 THEN 应该显示"点击跳转到样式定义"的工具提示

### 需求 5: 错误处理

**用户故事:** 作为开发者，我希望在跳转失败时能够得到清晰的错误信息，以便了解问题所在。

#### 验收标准

1. WHEN 样式定义不存在 THEN 系统应该显示"未找到样式定义: .class-name"的消息
2. WHEN 文件解析出错 THEN 系统应该显示"文件解析失败"的错误信息
3. WHEN 跳转操作失败 THEN 系统应该显示通用错误提示并记录详细错误日志
